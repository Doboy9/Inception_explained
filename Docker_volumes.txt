(Docker)volumes

Un volume Docker est un mécanisme de stockage persistant pour les conteneurs Docker. 
Contrairement aux systèmes de fichiers éphémères des conteneurs, qui disparaissent lorsque le conteneur est supprimé, les volumes permettent de conserver les données indépendamment du cycle de vie des conteneurs. 
Ils sont idéaux pour stocker des données critiques, comme les bases de données ou les fichiers générés par une application.

Il existe trois types de volumes : les volumes nommés, les bind mounts et les volumes anonymes.

Volume Nommé :

Un volume nommé est un volume Docker qui est créé et géré par Docker lui-même.
Il est donné un nom spécifique lors de sa création et peut être utilisé pour stocker des données persistantes entre différents conteneurs.
Stocké dans le répertoire Docker (/var/lib/docker/volumes/) sur la machine de l'utilisateur. 
C'est géré par Docker et l'utilisateur n'a pas besoin d'interagir directement avec ces fichiers.
Les volumes nommés ne sont pas detruit lors de l'arret des conteneurs. Ils sont detruitssi on lance la commande docker volume rm <volume>.

Bind Mount :

Un bind mount lie un répertoire spécifique de l'hôte à un répertoire dans le conteneur. Il permet un accès direct aux fichiers et répertoires de l'hôte.
Monté directement depuis un répertoire ou un fichier existant sur la machine de l'utilisateur (par exemple, /home/user/app), accessible et modifiable à la fois par le conteneur et l'utilisateur.
Le volume n'est detruit que si l'utilisateur le supprime de son pc.
Exemple de volume monté :
volumes:
  database:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /home/dboire/data/mysql -> Monte les volumes depuis notre pc depuis ce chemin

Volume Anonyme:

Un volume anonyme est créé automatiquement par Docker lorsqu'un répertoire est monté dans un conteneur sans spécifier de nom de volume. 
Ces volumes sont généralement utilisés pour stocker des données temporaires.
